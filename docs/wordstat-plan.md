## Этап 2 — идентификация неоднозначных ников ⬜

Для ников с ❓ нужны ссылки на YouTube/Twitch от пользователя (или подтверждение догадки),
прежде чем заводить профиль. Список вопросов — в разделе «Открытые вопросы» ниже.

## Этап 3 — поиск источников по каждой персоне ⬜

Правила (см. CLAUDE.md → Evidence model):
- ничего не добавляем в `equipment` без записи в `sources` с `url` + `checkedAt`;
- `confirmed` — только слова/видео самого блогера; иначе `reported`;
- устаревшая сборка (снятое с производства железо, старая платформа) — `historical`;
- нет ни одного проверяемого источника → `researchContent()` + `indexable: false` + `verificationStatus: 'research'`.

Приоритетные источники: HYPERPC / сборщики (проектные страницы), ProSettings.net (киберспорт),
PCstonks, Cybersport.ru, собственные видео-сетаптуры блогера.

## Этап 4 — наполнение `app/data/creators.ts` ⬜

Для каждой персоны: slug, aliases (обязательно кириллица — под запросы Вордстата),
двуязычный `content` (ru пишется нативно, не переводом), equipment, sources.

## Этап 5 — проверки ⬜

`npm run lint` → `npm run typecheck` → `npm run validate` → `npm run build` → `npm run smoke`.

### Что сработало при поиске источников

- **Каталог HYPERPC** `hyperpc.ru/project/hyperboost` — лучший источник по русским блогерам:
  проектные страницы с полным перечнем комплектующих. Оттуда взяты CheZee, Derzko69,
  «Держи дверь», EdisonPts (у него **две** страницы — ранняя и апгрейд), а раньше — Куплинов и Лололошка.
  Дат на страницах нет: свежее железо помечаем `reported`, устаревшее — `historical`.
- **ProSettings.net** — киберспорт (s1mple, m0NESY, donk, StRoGo, Recrent), есть дата обновления.

### Что не сработало

- WebSearch по русским нишевым запросам возвращает общие статьи «как собрать ПК для стрима», а не персоналии.
- **streamersbase.ru** — есть страницы почти на всех (windy31, drakeoffc, t2x2, deepins02, mellstroy),
  но **без дат**, с пометкой «данные собраны на основе открытых источников» и с явно устаревшими
  списками (у windy31 RTX 2060 рядом с Rampage VI Extreme). Ниже планки проекта — **не используем**.
- YouTube-описания через WebFetch не отдаются (страница рендерится JS), поэтому сборки,
  объявленные только в видео, автоматически не подтвердить — нужен ручной просмотр.
